# 프로젝트 개요: 업무 말투 변환기 (Business Tone Converter)

본 프로젝트는 사용자가 입력한 일상적인 말투의 문장을 수신자(상사, 동료, 고객 등)에 적합한 격식 있고 전문적인 업무용 어조로 변환해주는 AI 서비스 개발 프로젝트입니다.

## 📁 주요 문서 및 파일
- **개요서_업무말투변환기.md**: 프로젝트의 목적, 대상 사용자, 핵심 기능 및 전체 구조에 대한 고수준 요약.
- **PRD_업무말투변환기.md**: 상세 제품 요구사항 정의서. 기술 스택, 시스템 아키텍처, 단계별 구현 가이드 및 배포 방법 포함.
- **my-rules.md**: Gemini CLI 어시스턴트가 준수해야 할 시스템 지침 및 제약 사항 (한국어).

## 🛠 기술 스택 (예정)
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla JS)
- **Backend**: Python 3.11+, FastAPI, Uvicorn
- **AI/LLM**: LangChain, Upstage Solar-Pro2 API
- **Deployment**: Vercel (Frontend), Render/Railway (Backend)

## 🏗 시스템 아키텍처
1. **사용자**: 원문 입력 및 수신 대상 선택.
2. **프론트엔드 (Vercel)**: 백엔드 API 호출 (`POST /api/convert`).
3. **백엔드 (FastAPI)**: LangChain을 통해 Upstage Solar-Pro2 LLM 호출.
4. **AI (Upstage)**: 프롬프트 전략에 따라 말투 변환 처리 후 결과 반환.

## 📜 개발 원칙 (PRD 발췌)
1. **완료 기준을 먼저 정의하라**: "끝"의 기준이 명확해야 헤매지 않습니다.
2. **새 기술은 조사 먼저, 구현 나중**: 5분의 조사가 1시간의 디버깅을 아껴줍니다.
3. **버그는 분석 먼저, 수정 나중**: 원인 파악이 근본 해결로 이어집니다.
4. **배포 전 내가 한 번은 읽어라**: 코드를 이해해야 진정한 내 것이 됩니다.

## 🚀 향후 구현 단계 (STEP 1~4)
1. **STEP 1: 환경 준비**: 가상환경 설정, 디렉토리 구조 생성, API 키 설정.
2. **STEP 2: 백엔드 구현**: 데이터 모델 정의, 프롬프트 템플릿 작성, 변환 로직 구현.
3. **STEP 3: 프론트엔드 구현**: UI 레이아웃, 스타일링, API 연동 로직.
4. **STEP 4: 배포**: GitHub 푸시 및 Vercel/Render 배포.
